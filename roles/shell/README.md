# Shell Role

This role consolidates the configuration for various shell-related tools, including Zsh (with Oh My Zsh), Bash, Direnv, Starship, Btop, and more.

## Shell Initialization Files

Understanding the order and purpose of shell initialization files is crucial for correctly configuring your environment.

### Zsh (Z Shell)

Zsh reads its configuration files in the following order:

1.  **`.zshenv`**: Always read. Used for setting environment variables (like `PATH`) that should be available to all shells and scripts.
2.  **`.zprofile`**: Read only for login shells. It's the Zsh equivalent of `.bash_profile`.
3.  **`.zshrc`**: Read only for interactive shells. This is the main configuration file for Oh My Zsh, aliases, and interactive features.
4.  **`.zlogin`**: Read only for login shells, after `.zshrc`.
5.  **`.zlogout`**: Read when a login shell exits.

**Special Files:**
- **`.zcompdump`**: A cache file generated by the Zsh completion system (`compinit`) to speed up shell startup. This file is automatically created by Zsh.

---

### Bash (Bourne Again Shell)

Bash initialization depends on whether the shell is a "login shell" or an "interactive non-login shell":

1.  **Login Shells**:
    - Reads `/etc/profile` first.
    - Then looks for `~/.bash_profile`, `~/.bash_login`, or `~/.profile` (in that order) and reads the **first** one it finds.
    - Our configuration typically uses `~/.bash_profile` to source `~/.bashrc`.
2.  **Interactive Non-Login Shells**:
    - Reads `~/.bashrc`.
3.  **`.bash_logout`**: Read when a login shell exits.

---

## Configuration Guidelines

### Environment Variables (`PATH`, etc.)
- Use **`.zshenv`** for Zsh to ensure variables are available to both interactive shells and background scripts.
- For Bash, these are often defined in **`.bashrc`** or **`.bash_profile`**.

### Interactive Settings (Aliases, Prompts, Plugins)
- Use **`.zshrc`** for Zsh (Oh My Zsh is initialized here).
- Use **`.bashrc`** for Bash.

### Login-only Tasks
- Use **`.zprofile`** or **`.zlogin`** (Zsh) and **`.bash_profile`** (Bash) for tasks that should only run once when you first log in (e.g., starting an SSH agent if not already managed by the system).
